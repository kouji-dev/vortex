# Gateway Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the unified, governed LLM gateway: provider-compatible APIs (OpenAI / Anthropic / Bedrock), multi-provider routing, failover, rate limits, prompt caching, guardrails, observability, playground, eval.

**Architecture:** New module `server/api/src/ai_portal/gateway/` orchestrates an existing `catalog/` (providers + models) plus new sub-packages: `routing/`, `rate_limits/`, `cache/`, `guardrails/`, `traces/`, `playground/`, `evals/`. Provider-compatible endpoints live in `gateway/compat/openai.py`, `gateway/compat/anthropic.py`, `gateway/compat/bedrock.py` and translate to one canonical internal `LLMRequest`. All cross-cutting concerns (auth, audit, usage, budget, webhooks) come from `control_plane`.

**Tech Stack:** FastAPI, httpx + respx (testing), redis-py (cache), opentelemetry-api/sdk, tiktoken / anthropic-tokenizer / google-genai tokenizers, presidio-analyzer (PII), prompt-injection classifier (lightweight Llama-Guard-style via gateway-routed call optional).

**Spec:** `docs/superpowers/specs/2026-05-28-gateway-design.md`
**Depends on:** `2026-05-28-control-plane.md` (must merge first)

    tool_calls: list[ToolCall]

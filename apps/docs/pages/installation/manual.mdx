# Manual / Bare-metal

Install Vortex without Docker, directly on the host. Useful when Docker is unavailable or when you want more control over the environment.

import { Callout, Steps } from 'nextra/components'

## Requirements

- Python 3.12+
- PostgreSQL 15+ with pgvector extension ([install guide](https://github.com/pgvector/pgvector#installation))
- Node.js 20+ and pnpm
- (Optional) Redis 7+

<Steps>

### Set up PostgreSQL with pgvector

Create the database and enable the extension:

```sql
CREATE DATABASE ai_portal;
\c ai_portal
CREATE EXTENSION IF NOT EXISTS vector;
```

### Clone the repository

```bash
git clone https://github.com/your-org/vortex.git
cd vortex
```

### Create `config.yaml`

Place at `backend/config.yaml` (default location) or at a custom path set by `AI_PORTAL_CONFIG`:

```yaml
server:
  deployment_mode: selfhosted
  cors_origins: http://localhost:5173

database:
  url: postgresql+psycopg://postgres:yourpassword@127.0.0.1:5432/ai_portal

auth:
  secret_key: "your-secret-key-here"

smtp:
  host: smtp.example.com
  user: you@example.com
  password: "smtp-password"
  email_from: noreply@example.com

llm:
  anthropic_api_key: "sk-ant-..."

embedding:
  voyage_api_key: "pa-..."
```

Alternatively, export env vars directly — they take precedence over `config.yaml`. See [Configuration Reference →](/configuration/reference).

### Install and run the backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -e .

# Run migrations
alembic upgrade head

# Seed model catalog
seed-catalog-models

# Start the API
uvicorn ai_portal.main:app --host 0.0.0.0 --port 8000
```

<Callout type="info">
To use a config file at a custom path: `AI_PORTAL_CONFIG=/path/to/config.yaml uvicorn ...`
</Callout>

### Install and run the frontend

```bash
cd frontend
pnpm install
pnpm build
pnpm start          # production server on :3000
```

For development with hot reload:
```bash
pnpm dev --host     # starts on :5173, exposes on local network
```

### Run the setup wizard (selfhosted mode only)

Navigate to `http://localhost:3000/setup` and create your org and admin account.

</Steps>

## Running as a system service

Example `systemd` unit for the API (`/etc/systemd/system/vortex-api.service`):

```ini
[Unit]
Description=Vortex API
After=network.target postgresql.service

[Service]
Type=simple
User=vortex
WorkingDirectory=/opt/vortex/backend
Environment=AI_PORTAL_CONFIG=/opt/vortex/config.yaml
ExecStart=/opt/vortex/backend/.venv/bin/uvicorn ai_portal.main:app --host 0.0.0.0 --port 8000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Enable and start:
```bash
systemctl enable vortex-api
systemctl start vortex-api
systemctl status vortex-api
```

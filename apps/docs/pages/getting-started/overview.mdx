# Overview

Vortex is a self-hostable AI portal — a web application that gives your team a shared interface for chat, knowledge bases, and model governance.

## What it includes

- **Chat** — streaming conversations with any LLM (Anthropic Claude, OpenAI GPT, Google Gemini, or any OpenAI-compatible model)
- **Knowledge bases** — upload documents and query them via RAG (retrieval-augmented generation) backed by pgvector
- **Model catalog** — configure which models are available and set per-model defaults
- **Multi-user** — invite-only (`selfhosted` mode) or open signup (`saas` mode)

## Architecture

```
┌─────────────────────┐     ┌──────────────────────────┐
│  Browser (Frontend) │────▶│  FastAPI Backend (:8000)  │
│  React + TanStack   │     │  LangChain (in-process)   │
└─────────────────────┘     └──────────┬───────────────┘
                                        │
                           ┌────────────▼──────────────┐
                           │  PostgreSQL + pgvector     │
                           │  (chat history + vectors)  │
                           └───────────────────────────┘
                           ┌───────────────────────────┐
                           │  Redis (optional)          │
                           │  future background queues  │
                           └───────────────────────────┘
```

The backend calls LLM providers directly in-process via LangChain — no LiteLLM sidecar, no separate worker process. Document ingest runs on the API request path (thread-offloaded), not via a Redis queue.

# Backup & Restore

## What to back up

| Data | Location | Notes |
|---|---|---|
| Database | PostgreSQL `ai_portal` | Chat history, KB documents, vectors, users, orgs |
| Uploaded files | `UPLOAD_DIR` (default: `data/uploads`) | Raw knowledge base files |

All knowledge base vectors are stored in PostgreSQL — there is no separate vector store.

## Database backup

```bash
# Docker Compose setup
docker compose exec db pg_dump -U postgres ai_portal > backup-$(date +%Y%m%d).sql

# Manual / remote Postgres
pg_dump -h localhost -p 5434 -U postgres ai_portal > backup-$(date +%Y%m%d).sql
```

## File upload backup

```bash
# Copy uploads from the Docker volume
docker run --rm \
  -v ai-portal-uploads:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/uploads-$(date +%Y%m%d).tar.gz /data
```

## Restore

```bash
# Restore database
docker compose exec -T db psql -U postgres ai_portal < backup-20260415.sql

# Restore uploads
docker run --rm \
  -v ai-portal-uploads:/data \
  -v $(pwd):/backup \
  alpine tar xzf /backup/uploads-20260415.tar.gz -C /
```

## Automated backups

Use a cron job or a managed backup service (Supabase automatic backups, Render database backups, etc.) for scheduled snapshots.

Example cron (daily at 2am):
```
0 2 * * * pg_dump -h localhost -p 5434 -U postgres ai_portal | gzip > /backups/ai_portal-$(date +\%Y\%m\%d).sql.gz
```

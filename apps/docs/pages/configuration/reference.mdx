# Configuration Reference

Complete reference for all Vortex configuration options. Every option can be set via `config.yaml` or an environment variable — env vars take precedence.

import { Callout } from 'nextra/components'

<Callout type="info">
**Required** marks settings that must be set for production use. Settings marked with a deployment mode (e.g. "selfhosted/saas") are required only in that mode.
</Callout>

---

## `server`

Controls the API server bind address, CORS, file uploads, and deployment mode.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `server.host` | `API_HOST` | `0.0.0.0` | — | Bind address for the API server. |
| `server.port` | `API_PORT` | `8000` | — | TCP port for the API server. |
| `server.cors_origins` | `CORS_ORIGINS` | `http://localhost:5173` | **yes (prod)** | Comma-separated list of allowed frontend origins. Must include your frontend URL. |
| `server.upload_dir` | `UPLOAD_DIR` | `data/uploads` | — | Directory for uploaded knowledge base files. Relative to the working directory or absolute. |
| `server.deployment_mode` | `DEPLOYMENT_MODE` | `dev` | **yes (prod)** | One of `dev`, `selfhosted`, `saas`. Controls auth behavior. See [Auth & Deployment Modes →](/configuration/auth). |

## `database`

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `database.url` | `DATABASE_URL` | `postgresql+psycopg://postgres:postgres@127.0.0.1:5434/ai_portal` | **yes (prod)** | PostgreSQL connection string using the `psycopg` (v3) driver. The target database must have the **pgvector extension** enabled. |

<Callout type="warning">
Inside Docker containers, use the service hostname (`db`) not `127.0.0.1`. Example: `postgresql+psycopg://postgres:postgres@db:5432/ai_portal`.
</Callout>

## `auth`

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `auth.mode` | `AUTH_MODE` | `dev` | — | Legacy auth mode selector: `dev` or `entra`. For new deployments, use `server.deployment_mode` instead. |
| `auth.secret_key` | `SECRET_KEY` | `` | **selfhosted / saas** | JWT signing secret. Generate: `python -c "import secrets; print(secrets.token_hex(32))"`. Never use the empty default in production. |
| `auth.dev_bearer_token` | `DEV_BEARER_TOKEN` | `devtoken` | — | Fixed bearer token accepted in `dev` mode. **Never use in production.** |
| `auth.dev_seed_user_email` | `DEV_SEED_USER_EMAIL` | `dev@localhost` | — | Email of the auto-created seed user in `dev` mode. |
| `auth.portal_api_key_pepper` | `PORTAL_API_KEY_PEPPER` | `` | **if AUTH_MODE=entra** | HMAC pepper for portal API keys (`aip_…`). Set a long random secret in production. |
| `auth.entra_tenant_id` | `ENTRA_TENANT_ID` | `` | if AUTH_MODE=entra | Microsoft Entra directory (tenant) UUID. |
| `auth.entra_api_audience` | `ENTRA_API_AUDIENCE` | `` | if AUTH_MODE=entra | JWT `aud` claim — e.g. `api://<client-id>` or Application ID URI. |
| `auth.entra_debug_jwt` | `ENTRA_DEBUG_JWT` | `false` | — | Include PyJWT error text in 401 responses. Local debugging only — never enable in production. |

## `smtp`

Required in `selfhosted` and `saas` modes for email verification, password reset, and invites.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `smtp.host` | `SMTP_HOST` | `` | **selfhosted / saas** | SMTP server hostname. |
| `smtp.port` | `SMTP_PORT` | `587` | — | SMTP port. 587 = STARTTLS, 465 = SSL/TLS. |
| `smtp.user` | `SMTP_USER` | `` | — | SMTP authentication username. |
| `smtp.password` | `SMTP_PASSWORD` | `` | — | SMTP authentication password. |
| `smtp.email_from` | `EMAIL_FROM` | `noreply@example.com` | — | From address for outgoing emails. |

## `llm`

API keys and defaults for language model providers. At least one key must be set for chat to work.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `llm.openai_api_key` | `OPENAI_API_KEY` | `` | one LLM key required | API key for OpenAI or any OpenAI-compatible endpoint. |
| `llm.openai_api_base` | `OPENAI_API_BASE` | `https://api.openai.com/v1` | — | Base URL for OpenAI-compatible API. Change to route to Azure OpenAI, a local model server (LM Studio, Ollama), or any proxy. |
| `llm.anthropic_api_key` | `ANTHROPIC_API_KEY` | `` | one LLM key required | API key for Anthropic Claude models. |
| `llm.gemini_api_key` | `GEMINI_API_KEY` | `` | one LLM key required | API key for Google Gemini models. |
| `llm.chat_default_api_model` | `CHAT_DEFAULT_API_MODEL` | `gemini-2.5-flash-lite` | — | Default model vendor API ID for new conversations and streaming fallback when no model is set. Also accepted: `CHAT_DEFAULT_MODEL`, `CHAT_MODEL` (legacy aliases). |
| `llm.default_system_prompt` | — | `You are a helpful assistant.` | — | Default system prompt applied to all conversations that don't set their own. |
| `llm.user_search_country` | `USER_SEARCH_COUNTRY` | `FR` | — | ISO 3166-1 alpha-2 country code passed to web search providers to localize results. |

## `embedding`

Controls vector embeddings for knowledge base ingestion and RAG. Required if using knowledge bases.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `embedding.voyage_api_key` | `VOYAGE_API_KEY` | `` | **yes (knowledge bases)** | Voyage AI API key. When set, Voyage is used for embeddings (recommended). |
| `embedding.model` | `EMBEDDING_MODEL` | `` (empty) | — | Embedding model name. Empty = auto-select: `voyage-4-lite` when `VOYAGE_API_KEY` is set, `text-embedding-3-small` (at 1024 dimensions) when using OpenAI. Set explicitly to use a different model. |

<Callout type="warning">
**Do not switch embedding models after ingesting documents.** Vectors from different models are not comparable. If you change providers, delete and re-ingest all knowledge base documents.
</Callout>

## `ingest`

Controls document ingest behavior for knowledge bases.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `ingest.max_file_size_mb` | `KB_MAX_FILE_SIZE_MB` | `500` | — | Maximum upload size per knowledge base file in MB. |
| `ingest.commit_batch_size` | `INGEST_COMMIT_BATCH_SIZE` | `100` | — | Chunks committed per DB transaction during ingest. Lower values reduce memory pressure on large files. |
| `ingest.embed_batch_size` | `INGEST_EMBED_BATCH_SIZE` | `128` | — | Chunks sent to the embedding API per request. Lower values reduce API timeout risk. |

---

## `rag`

Retrieval-Augmented Generation parameters. Controls how many chunks are retrieved and filtered.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `rag.max_top_k` | `RAG_MAX_TOP_K` | `30` | — | Maximum number of chunks retrieved per RAG query. |
| `rag.min_top_k` | `RAG_MIN_TOP_K` | `8` | — | Minimum chunks retrieved even if the similarity threshold filters more out. Ensures a floor of context. |
| `rag.similarity_threshold` | `RAG_SIMILARITY_THRESHOLD` | `0.3` | — | Cosine similarity cutoff (0–1). Chunks scoring below this are discarded. Lower = more results, potentially noisier. |
| `rag.max_tool_iterations` | `RAG_MAX_TOOL_ITERATIONS` | `1` | — | Maximum RAG tool call iterations per chat turn. |

---

## `conversation`

Controls how conversation history is managed to keep context window usage bounded.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `conversation.base_window_size` | `CONVERSATION_BASE_WINDOW_SIZE` | `30` | — | Number of messages kept in the sliding context window before older messages are summarized. Alias: `CONVERSATION_WINDOW_SIZE`. |
| `conversation.summary_interval` | `CONVERSATION_SUMMARY_INTERVAL` | `10` | — | Trigger a rolling summary every N new messages. |
| `conversation.inactivity_summary_hours` | `CONVERSATION_INACTIVITY_SUMMARY_HOURS` | `1` | — | Trigger a summary after this many hours of inactivity in a conversation. |

## `observability`

Optional LLM tracing via [Langfuse](https://langfuse.com). When keys are set, all LLM calls are traced.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `observability.langfuse_public_key` | — | `` | — | Langfuse public key. Leave empty to disable tracing. |
| `observability.langfuse_secret_key` | — | `` | — | Langfuse secret key. |
| `observability.langfuse_host` | — | `https://cloud.langfuse.com` | — | Langfuse server URL. Override to point at a self-hosted Langfuse instance. |

## `search`

Configures the web search provider for the `web_search` tool available to users in chat.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `search.provider` | `SEARCH_PROVIDER` | `duckduckgo` | — | Web search provider. Options: `duckduckgo`, `tavily`, `serper`, `exa`. If a provider is set but its API key is missing, Vortex automatically falls back to DuckDuckGo. |
| `search.tavily_api_key` | `TAVILY_API_KEY` | `` | if provider=tavily | Tavily Search API key. |
| `search.serper_api_key` | `SERPER_API_KEY` | `` | if provider=serper | Serper (Google Search) API key. |
| `search.exa_api_key` | `EXA_API_KEY` | `` | if provider=exa | Exa semantic search API key. |

---

## `fetch`

Configures fetch providers for the `fetch_webpage` tool. Providers are tried in order — first success wins.

| YAML key | Env var | Default | Required | Description |
|---|---|---|---|---|
| `fetch.firecrawl_api_key` | `FIRECRAWL_API_KEY` | `` | — | Firecrawl API key. When set, Firecrawl is placed first in the fetch chain (best Cloudflare/bot-protection bypass). |
| `fetch.jina_api_key` | `JINA_API_KEY` | `` | — | Jina Reader API key. Jina is always in the chain (free tier available without a key); a key raises rate limits. |

See [Fetch Providers →](/configuration/fetch) for the full provider chain and fallback behavior.

# Embeddings

Embeddings are required for knowledge base ingestion and RAG (retrieval-augmented generation). Without an embedding provider, document uploads will fail.

import { Callout } from 'nextra/components'

## How embeddings work

When a document is uploaded to a knowledge base:
1. The text is split into chunks
2. Each chunk is sent to the embedding provider to generate a vector
3. Vectors are stored in PostgreSQL via pgvector (`vector(1024)`)

At query time, the user's question is embedded and compared against stored vectors using cosine similarity.

## Option A — Voyage AI (recommended)

Set `VOYAGE_API_KEY`. No other configuration needed.

```yaml
embedding:
  voyage_api_key: "pa-..."
```

Vortex uses `voyage-4-lite` by default:
- 1024-dimensional vectors (matches the pgvector column)
- **200M tokens/month free** on the Voyage free tier
- Lowest-cost Voyage text embedding model

To use a different Voyage model:
```yaml
embedding:
  voyage_api_key: "pa-..."
  model: voyage-4
```

## Option B — OpenAI-compatible

If `VOYAGE_API_KEY` is not set, Vortex falls back to OpenAI-compatible embeddings using `OPENAI_API_KEY`:

```yaml
llm:
  openai_api_key: "sk-..."
  openai_api_base: https://api.openai.com/v1    # or compatible endpoint

embedding:
  model: text-embedding-3-small    # optional; this is the default
```

Vortex requests 1024 dimensions from OpenAI models to match the pgvector column.

## Choosing a provider

| | Voyage AI | OpenAI |
|---|---|---|
| Free tier | 200M tokens/month | Limited (pay-as-you-go) |
| Setup | One key (`VOYAGE_API_KEY`) | Uses your `OPENAI_API_KEY` |
| Default model | `voyage-4-lite` | `text-embedding-3-small` |
| Recommendation | ✅ Preferred | ✅ Good if already using OpenAI |

<Callout type="warning">
**Do not switch embedding providers after ingesting documents.**

Vectors from different models are not comparable. If you switch from Voyage to OpenAI (or vice versa), you must delete and re-ingest all knowledge base documents. The vectors stored for old documents will return incorrect similarity scores.
</Callout>

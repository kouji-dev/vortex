# Auth Plan

**Rule:** the app never logs anyone in. It only trusts OIDC tokens from an IdP.
The app owns orgs, teams, invites, RBAC.

## SaaS IdP choice

Free solution: **Authlib + FastAPI sessions** (self-owned).
Scope: username/password + GitHub + Google social.

## One IdP, both modes

```
SaaS:        your IdP (signup + social)   ─┐
                                            ├─ OIDC token ─► YOUR APP
Enterprise:  customer's Keycloak  ─────────┘              (orgs, invites, RBAC)
```

Same app code. Only the IdP changes.

## Auth logic = 4 pieces (build once)

1. **Validate token** — verify JWT signature against IdP's JWKS.
2. **JIT-provision** — first login → create user + org membership in your DB.
3. **Map `groups` claim → your RBAC role.**
4. **Enforce RBAC** — your code, per request.

## What differs per mode

| | SaaS | Enterprise |
|---|---|---|
| IdP | Yours (Authlib + FastAPI) | Theirs (Keycloak) |
| Signup / social | You enable | They own |
| Invites / orgs | **Your app** | **Your app** |
| RBAC | **Your app** | **Your app** |
| Config | point OIDC at your IdP | point OIDC at their IdP |

## Invites (SaaS) — app's job, not the IdP

- User clicks invite → app creates pending membership + emails link.
- Invitee signs up via IdP → JIT-provision attaches them to the org.

# Refactor Actions

Scope: **auth only**. Every action either builds the OIDC-consumer flow or **removes** auth code/config that doesn't serve SaaS or the first enterprise (Keycloak) path.

One config flag drives everything: `DEPLOYMENT_MODE = saas | enterprise` (already in `render.yaml`).

## A. Core — OIDC consumer (build once, both modes)

- [ ] Add `auth/oidc/consumer.py` — validate JWT against IdP JWKS (cache JWKS, check `iss`/`aud`/`exp`).
- [ ] Add `auth/oidc/provisioning.py` — JIT: first valid token → create user + org membership.
- [ ] Add `auth/oidc/role_map.py` — map token `groups` claim → internal RBAC role (config per IdP).
- [ ] Single FastAPI dependency `require_user()` → validate + provision + attach RBAC. Use everywhere.
- [ ] Keep existing `rbac/` evaluator — RBAC stays ours, unchanged.

## B. SaaS mode (Authlib + FastAPI sessions)

- [ ] `auth/saas_idp/` — Authlib: username/password + GitHub + Google.
- [ ] Issue our own signed JWT (same shape the OIDC consumer expects → core stays IdP-agnostic).
- [ ] Org create + **invite** flow: pending membership + email link → JIT on signup.
- [ ] Env: `OAUTH_GITHUB_ID/SECRET`, `OAUTH_GOOGLE_ID/SECRET`, `JWT_SIGNING_KEY`.

## C. Enterprise mode (customer Keycloak)

- [ ] Reuse `auth/idp/providers/oidc.py` — point OIDC consumer at customer issuer URL.
- [ ] Config: `OIDC_ISSUER`, `OIDC_CLIENT_ID`, `OIDC_CLIENT_SECRET`, `OIDC_GROUP_ROLE_MAP`.
- [ ] No signup/social — login redirects to their IdP. Invites optional (their directory is source of truth).
- [ ] Audit which hand-rolled providers to **delete** (saml/okta/entra/ldap/scim) until a client needs them.
